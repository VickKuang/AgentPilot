# 模块进度文档（MVP）

> 更新时间：2026-04-28
> 
> 说明：本进度基于当前仓库实现状态梳理，按“已实现 / 未实现（或仅占位）/ 建议下一步”分项列出。

## 1. 任务管理模块

**进度：100%（已完成）**

### 已实现
- 任务创建、查询列表、查询详情 API。
- 任务补参（支持 JSON merge）与必填参数缺失检测。
- 敏感字段自动脱敏（`password/token/secret`）。
- 任务状态流转：`pending/running/paused/passed/failed/blocked/terminated`。
- 任务状态机约束（统一迁移矩阵，非法迁移直接返回错误）。
- 任务列表支持分页、过滤、排序（`page/page_size/status/sort_by/sort_order`）。

### 未实现 / 待完善
- 无（MVP 任务1已完成）。

### 建议下一步
- 为任务列表补充组合条件检索（按时间区间 + 关键字）。
- 增加列表返回字段裁剪（summary 视图）以降低大列表负载。

## 2. 执行编排（Planner → Observer → Action → Verifier）模块

**进度：100%（流程闭环完成）**

### 已实现
- 基于用户目标的场景规划器：登录、搜索、表单、筛选、异常提示、通用场景。
- 执行管线支持：启动、重试、暂停、恢复、终止。
- 支持步骤级重试、全局超时与步骤超时。
- 执行过程中可写入步骤日志、工具调用日志、页面快照。
- Observer 输出改为场景化页面结构（按页面返回不同元素），Action 输出结构化执行消息。
- Verifier 增加可配置规则（元素存在 / 文本包含 / 页面匹配）并按步骤断言。
- 暂停/恢复支持断点续跑语义：从“上次成功 step + 1”继续执行。

### 未实现 / 待完善
- 尚未接入真实浏览器/移动端驱动（Playwright/WebDriver/Appium）。
- 截图 URL 仍为模拟路径，待对接真实对象存储。

### 建议下一步
- 引入真实执行驱动并与当前编排状态机打通。
- 在断言规则中继续扩展属性、可见性、网络响应等能力。

## 3. 可观测性模块（日志/快照/进度）

**进度：100%（任务3已完成）**

### 已实现
- 步骤日志查询 API。
- 工具调用日志查询 API（含请求/响应/耗时）。
- 页面快照查询 API。
- 执行进度 API（总步骤、完成数、成功/失败数、百分比）。
- 步骤日志支持按 `step_order/status/started_at/ended_at` 查询筛选。
- 工具调用日志支持按 `step_order/success/tool_name/started_at/ended_at` 查询筛选。
- 页面快照支持按 `step_order/current_page/started_at/ended_at` 查询筛选。
- 新增失败聚合视图 API：`/api/v1/tasks/:task_id/logs/failures`（失败步骤数、步骤编号、步骤名称、失败工具、最近失败时间）。

### 未实现 / 待完善
- 快照仍记录 mock 路径，缺少真实静态资源落盘与可访问链接。

### 建议下一步
- 对接对象存储并输出可打开的截图 URL。

## 4. 报告模块（测试报告 / Bug 报告 / 多格式导出）

**进度：100%（已完整实现）**

### 已实现
- 测试报告 API。
- Bug 报告 API（失败时自动生成严重级别与复现步骤）。
- Markdown/HTML/PDF 报告导出 API（同一导出端点按 `format` 参数切换）。
- 报告导出模板化（支持默认模板 + 自定义模板占位符渲染）。
- Bug 严重级别可配置评估策略（结合场景、失败步数、重试次数、超时原因生成 P1/P2/P3）。
- 报告汇总包括：结果、摘要、步骤、截图列表等。

### 未实现 / 待完善
- 暂未接入品牌化模板市场（当前为单模板字符串渲染）。
- PDF 暂为轻量文本版导出，后续可补图表与样式排版。

### 建议下一步
- 抽象模板仓库（按项目/租户维度管理模板版本）。
- 增加富文本 PDF 样式引擎与企业 Logo/页眉页脚配置。

## 6. 存储与数据层模块

**进度：100%（模块目标已落地）**

### 已实现
- 存储层已抽象为可切换实现：默认本地 `data/store.json`，设置 `DATABASE_URL` 后自动切换 PostgreSQL。
- PostgreSQL 持久化已在后端真实接入，支持启动加载与运行期持续写入。
- 写入路径引入事务边界与乐观并发控制（版本号 + 冲突重试）。
- 新增审计字段（`created_at` / `updated_at`）用于存储状态追踪。

### 未实现 / 待完善
- 目前 PostgreSQL 为“状态快照表”模式，后续可按业务表做更细粒度拆分（任务/步骤/日志分表读写）。
- 跨实例高并发场景建议补充压测与监控告警阈值。

### 建议下一步
- 逐步将快照式存储演进为领域 Repository（按任务、日志、报告拆分）。
- 为历史数据引入归档策略（冷热分层 + 对象存储）。

## 7. 安全与平台能力模块

**进度：90%（MVP 核心能力已完成）**

### 已实现
- API Key 鉴权中间件（`x-api-key`），支持凭证从环境变量加载。
- RBAC（`admin/operator/viewer`）按 HTTP 方法进行权限控制。
- 多租户隔离（`x-tenant-id`）：任务、报告、日志查询均按租户约束。
- API 限流（按 key + tenant + method 的分钟窗口计数）。
- 审计日志（任务创建、启动、重试、暂停、恢复、终止、补参）与审计查询 API。
- 密钥管理集成：凭证元信息包含 `kms_key_ref`，并提供安全上下文 API 返回 `kms_provider` / `key_id`。
- 保留敏感字段脱敏存储能力。

### 未实现 / 待完善
- 当前鉴权方案为 API Key，尚未支持 JWT/OIDC 与短期令牌轮换。
- 限流为单节点内存计数，未接入 Redis/网关级全局限流。
- 审计日志尚未做不可篡改归档与告警联动。

### 建议下一步
- 增加 JWT + API Key 双栈鉴权，并支持密钥轮换与吊销。
- 审计日志接入外部 SIEM，增加风险操作告警。

## 总体结论（当前版本定位）

- 当前代码已达到 **“MVP 演示闭环”**：能创建任务、执行模拟流程、查看日志与报告。
- 若目标是 **生产可用自动化测试平台**，主要缺口集中在：
  1) 真实执行引擎，2) 生产级存储，3) 安全与多租户，4) 实时通信，5) 集成生态与 CI 质量体系。
- 结合现状建议定义里程碑：
  - **M1（可试点）**：真实驱动 + PostgreSQL + API 鉴权。
  - **M2（可团队级使用）**：WebSocket + 完整测试体系 + 报告模板化。
  - **M3（平台化）**：多租户 + 外部系统集成 + 审计合规。
